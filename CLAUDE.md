# Slime Polish

Godot 4.6 製の2Dゲーム（1280x720）。画面フロー: タイトル → キャラ選択 → オープニング（初回のみ） → 磨き画面 → リザルト。

## 構成

- `godot/` — Godot プロジェクト本体（`project.godot`、メインシーンは `scenes/main.tscn`）
  - `scripts/game_rules.gd` — 純粋なゲームルール計算（Node 非依存、ユニットテスト対象）
  - `scripts/game_screen.gd` — 磨き画面の進行・ゲージ・FINISH/失敗・表情
  - `scripts/game_screen_brushes.gd` — ブラシの入力・ドラッグ・アンロック・衝突補正
  - ファイルごとの担当一覧は `docs/SPEC.md` 参照
- `docs/` — `SPEC.md`（構造の地図）、`asset_list.md`（素材発注書）、`bgm_credits.md`（曲対応・ライセンス）

## テスト

```
./scripts/test.sh   # 全テスト（ルール + フロー）
```

失敗時は非ゼロで終了。ゲームロジックを変えたら実行すること。
`godot` は WSL の PATH 上の 4.6 バイナリ（`~/.local/bin/godot`）。

## 設計方針

**コードとデータを最初から分離する。** 「コードを読まない人が編集したくなるもの」
（セリフ・プロフィール等のテキスト、座標、バランス数値、アセットパス、対応表）は
main.gd 等のロジックに直書きせず、最初から専用のデータファイルに置く。

- キャラ関連のデータ → `godot/data/characters/*.tres`（形式と読込は `scripts/character_definition.gd` / `characters.gd`）
- ゲームバランス定数 → `godot/scripts/game_rules.gd`
- 素材の対応・出典 → `docs/`（例: `bgm_credits.md`）
- 新しい種類のデータが増えるときは、main.gd に足す前に置き場を決める

## ドキュメント方針

**コードが唯一の真実。docs はコードの写しを持たない。**

- `docs/SPEC.md` は「構造の地図」だけ（ファイルの担当・画面フロー・セーブ形式・テスト手順）。
  **数値・数式・ブラシ等の一覧・挙動の詳細を書き足すことを禁止**（コードに書く）。
  実装変更のたびに SPEC を追従修正している気配が出たら、それは書きすぎのサイン
- `docs/asset_list.md` = 素材の発注書。**スロットの仕様（パス・サイズ・制約）だけを書き、
  「配置済み」「現素材は◯◯」等の状態を書かない**（状態はファイルシステムとコードが真実）
- `docs/bgm_credits.md` = 出典・ライセンス記録。素材を入れ替えたときだけ更新する
- 役目を終えた文書は残さず削除する（git履歴が保存してくれる）

## ポリシー
