# アセット発注書

> **このドキュメントは「スロットの仕様」だけを書く。**
> 各素材の置き場所・サイズ・形式・構図の制約のみ。
> 「配置済み」「現素材は◯◯」等の状態は書かない（ファイルシステムとコードが真実）。
> 曲・フォント等の出典/ライセンスの記録は `bgm_credits.md`。

- 置き場所はすべて `godot/` 配下。画像はPNG、音はOGG推奨（ogg/wav/mp3対応）。
- ファイル名は小文字スネークケース。キャラidは `general`（女将軍）/ `admiral`（エルフ提督）。

## キャラ表情立ち絵（2人 × 11表情）

`godot/assets/chara/<キャラid>/<表情id>.png`。置くだけで自動反映、無い表情はプレースホルダ表示。
表情idの一覧と発生条件は `expression_rules.gd`、原案は `表情差分と音.xlsx`。

- 磨き画面の立ち絵と同じ枠（640×720・cover表示）で差し替わるため、**下記の磨き画面立ち絵と同一キャンバス・同一立ち位置**にすること
- まず `idle_a` / `touch_c` / `climax` の3枚で表示確認してから残りを差分制作するのがおすすめ

## 磨き画面の立ち絵ベース

`godot/assets/chara/<キャラid>/game_background.png`。

- **正方形（800×800〜1024×1024）の顔と胸のバストアップ**
- 640×720領域に天地いっぱい・左右クロップ（cover）で表示される。正方形なら左右約11%ずつ切れるので、顔と胸を中央に置き、左右端に重要な要素を置かない
- 差し替えたら磨きターゲット座標の再計測が必要（`characters.gd` の換算式コメント参照）

## キャラ選択ポートレート（2人 × 2状態）

`godot/assets/chara/<キャラid>/portrait.png` / `portrait_after_opening.png`。
縦長推奨。アスペクト維持でカード内に収まる最大サイズで中央表示される。
オープニング画面の左半分にも共用されるため、オープニング専用CGは不要。

## リザルト画面立ち絵（2人 × 1枚）

`godot/assets/chara/<キャラid>/result.png`。置くだけで自動反映、無ければ非表示。
リザルト画面右側に960×1280相当の縦長比率（3:4）で表示される。

## 磨きターゲット画像（2人 × 左右 = 4枚）

乳首部分のオーバーレイ小画像。`characters.gd` の `left/right` の `image` にパスを書く。
ターゲットの直径（半径×2。現在は80px）の円形領域に伸縮 → **128×128の透過PNG**。
立ち絵と同じ拡大率で切り出すとなじむ。
表情差分で立ち絵が切り替わっても同じ位置に載り続ける想定
（表情ごとに乳首の位置や状態が変わる絵にする場合は事前に相談）。

## おっぱいレイヤー（キャラごと × 左右 = 2枚）

乳首の動きに胸がついてくる演出用。ターゲット（乳首）が引っ張りで最大48px動き、
胸レイヤーはその何割かを遅れて追従する。画像が無いキャラは従来どおり1枚絵表示。

`godot/assets/chara/<キャラid>/breast_left.png` / `breast_right.png`（left=画面左）。

- **立ち絵ベースと同一キャンバスサイズの透過PNG**（切り出し元と同じ座標に置いたまま書き出す。
  位置合わせはコード側で不要になる）
- 各ファイルは**その胸だけ**を含める。**乳首・乳輪は含めない**（磨きターゲット画像スロットに分離）
- 谷間で重なる部分は**奥側の胸も隠れている所まで完全に描く**（動いたとき欠けないように）。
  左右どちらを手前に重ねるかを決めて伝えること
- このスロットを使うキャラは、立ち絵ベース（`game_background.png`）と全表情差分を
  **胸なし＋胸の裏側（体・服・床）を塗り足した絵**にする。胸は50px程度動いても
  破綻しない範囲まで裏を描いておけば十分

## ブラシ画像（9枚＋アクション差分）

`godot/assets/brushes/<ブラシid>.png`。置くだけで自動反映（無ければ従来の円表示）。
ブラシid: finger（指）/ tongue（舌）/ feather（羽）/ fude（筆）/ teeth（歯）/
toothbrush（歯ブラシ）/ rotary（回転ブラシ）/ tawashi（たわし）/ candle（ろうそく）。
`<ブラシid>_pinch.png` を置くと、つまみアクション中だけその画像に切り替わる（現在は finger 用）。

- **128×128の透過PNG**目安。画像の最大辺が当たり判定の直径（全ブラシ共通128px、等倍表示）に合わせて縮小される
- 当たり判定は円なので、絵の実体をキャンバスいっぱいの円形領域に収めるとズレが少ない
- rotary は置いている間ずっと回転する。回転して見栄えのする（点対称すぎない）絵にすると効果的

## ハートパーティクル

`godot/assets/fx/hearts/*.png`。快感優勢時に舞う常時パーティクルとFINISHバーストの両方で、
このフォルダの中から毎回ランダムに1枚を選んで使う（`fx_textures.gd`）。

- 透過PNG、最大辺128px目安
- 色は絵自体が持つ色をそのまま使う（コード側で色を掛けない）ので、塗り済みの絵を置くこと
- 枚数を増減したら `fx_textures.gd` の `HEART_FILES` を合わせて更新する

`godot/fonts/` に置き、`theme/main_theme.tres` の参照を差し替える。
**配布前にライセンス（再配布可否・表記義務）を確認し、必要なら許諾文を同梱すること。**

## 音

置き場所・id・自動ロード規則は `game_audio.gd` の冒頭コメントが真実。要点だけ:

- BGM: `assets/audio/bgm/<id>`（画面ごとのid体系。曲の対応と出典は `bgm_credits.md`）
- SE 9種: `assets/audio/se/<id>` — brush_soft / brush_mid / brush_strong / brush_pain（接触ループ、
  ループ素材で）、climax（FINISH演出5秒に合わせる）、despair（失敗暗転2.5秒）、heartbeat（ループ）、
  ui_click、levelup（リザルトのレベルアップ演出、1〜1.5秒のファンファーレ系）
- ボイス: `assets/audio/voice/<キャラid>/<表情id>` — 表情と同じid体系。あるぶんだけ鳴る。
  候補素材は `/voice_candidates/` に退避中（git管理外）
